## 3. Missing Notebooks (Created)

The README mentions notebooks 01-09, but three critical research areas were not covered:

### 3.1 Notebook 10: Multi-Region CTI Calibration

**Research Question:**  
Are the current Composite Threat Index (CTI) thresholds optimal for each region?

**Why it matters:**
- `campaigns.csv` has `target_regions` (Estonia, Latvia, Lithuania, Baltic)
- Current system likely uses global thresholds (HIGH/MEDIUM/LOW)
- Regional threat baselines may differ (e.g., Estonia receives more airspace violations)

**Created:** `10_multi_region_cti_calibration.py`

**Key tasks:**
- Segment campaigns by `target_regions`
- Compute ROC curves per region
- Recommend region-specific thresholds
- Measure precision/recall improvement

### 3.2 Notebook 11: Campaign Lifecycle Analysis

**Research Question:**  
How long do disinformation campaigns last? What are the decay curves?

**Why it matters:**
- Informs collector retention policies (how long to keep signals?)
- Predicts when campaigns become "cold" (no longer actively spreading)
- Distinguishes "burst" campaigns (hours) from "slow burn" (days/weeks)

**Created:** `11_campaign_lifecycle_analysis.py`

**Key tasks:**
- Compute campaign duration from `cluster_members` timestamps
- Fit exponential decay models to signal volume over time
- Calculate half-life (time for 50% signal dropoff)
- Cluster campaigns by lifecycle pattern

### 3.3 Notebook 12: Sampling Frequency Optimization

**Research Question:**  
Are collectors sampling fast enough for each signal type? (Nyquist analysis)

**Why it matters:**
- Over-sampling wastes API quota (e.g., polling RSS every 5min when updates are hourly)
- Under-sampling misses fast-moving events (e.g., Telegram bursts)
- Trade-off between cost and detection latency

**Created:** `12_sampling_frequency_optimization.py`

**Key tasks:**
- Compute inter-arrival time distributions by `source_type`
- Calculate Nyquist rate (2× dominant frequency)
- Compare against current collector intervals
- Recommend optimal sampling rates per source
